## Root Cause Analysis: The Missing Link

### LevelVisualBridge NOT in Scene

**File** : `Assets/Scripts/Visual/LevelVisualBridge.cs`

**Class** : Regular `MonoBehaviour` (NOT MonoSingleton)

**Responsibility** : Orchestrates initialization of Weather + SceneDecor + PathTiles on `LevelEvents.OnLevelStart`

**Current Status** : ❌ **NOT added to Main.unity**

**What should happen**:
```csharp
private static void HandleLevelStart(Data.LevelData level, Bounds gridBounds)
{
    WeatherController.Instance?.ApplyTheme(theme);
    SceneDecor.Instance?.SpawnForLevel(theme, level.Id, gridBounds);
    PathTiles.Instance.BuildForLevelProgressive(grid, theme, spawnPos);
}
```

**What actually happens** : OnLevelStart fires, but LevelVisualBridge is never enabled, so HandleLevelStart never runs. Weather/SceneDecor/PathTiles remain uninitialized.

## Code Inspection Findings

### Scene Analysis
- **Main.unity** : Contains VfxPool, ThemeAmbientController, Castle, CameraController, JuiceFX
- **Missing** : LevelVisualBridge (critical), WeatherController (auto-creates ok), SceneDecor (auto-creates ok), DynamicEventManager (auto-creates ok)
- **Implication** : Orchestration layer (LevelVisualBridge) absent → downstream systems never subscribe → initialization never fires

### MonoSingleton Auto-Create Pattern
- WeatherController, SkyboxController, SceneDecor, DynamicEventManager all use `MonoSingleton<T>` pattern
- Auto-create instantiates GO if missing, but **OnEnable must fire for subscription**
- If auto-created GO is disabled (unlikely), subscriptions fail silently

### Asset Status
- Textures : Assets/Textures/ exists with subdirs (Anim, Sky, Ground, VFX, Tiles, etc.) but many PNG files missing
- Prefabs : Prefabs/Weather/, Prefabs/Towers/, Prefabs/Enemies/ exist with partial content
