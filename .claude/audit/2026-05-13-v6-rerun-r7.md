## Top 5 Remaining Gaps (R8 Sprint Candidates)

### Gap 1 — Audio SFX mismatches (A-12, A-13, A-14) — **P0**

`step_dirt` plays `hero_shoot.ogg` (gunshot on every enemy step, 100+/wave). `cancel`/`menu_*`/`tower_*_click` all play `boom.ogg` (explosion for UI clicks). `hero_death`→boss roar, `hero_ult`→basic shoot. No new recordings required for quick wins: remap `xp_pickup`→`gem_gain.ogg`, `tower_sold`→`tower_upgrade.ogg`, `step_dirt` needs real footstep or silence. Effort: 30 min remapping + 6 real clips (P1 asset work).

### Gap 2 — Animator state machines runtime unverified (P-07) — **P1**

102 `.controller` files present in Resources/Animations/Controllers. `c32ac3c` wired idle+walk. No Play mode validation since UnityMCP was blocked at time of wiring. T-pose risk remains for all 28 enemy types + hero. Requires Unity Editor open session + `run_play_mode` verification. Effort: 1-2h.

### Gap 3 — Castle real art .fbx absent (V-06, V-07) — **P1**

4 procedural meshes are placeholders. No `.fbx` files exist anywhere in Assets. SkinRegistry has 10 castle skin entries but they all reference procedural geometry. Visual gap is user-visible (castle looks like a gray cube cluster). Effort: Asset creation/import + CastleSkinController wiring, 2-3h.

### Gap 4 — Water/lava tile animation unverified runtime (V-03, V-04) — **P2**

8 PNG frames present in Resources/Tiles/Water/ and Resources/Tiles/Lava/. WaterLavaAnimController loads via `Resources.Load("Tiles/Water/water_frame_{i:D2}")`. Code complete but no Play mode confirm that tile MeshRenderers are registered and `_texturesLoaded=true`. Fallback to colour palette in case of miss. Effort: 30 min runtime verify via Unity-MCP.

### Gap 5 — L.cs localization (47k LOC hardcoded) (P-06) — **P3**

L.cs contains all en/fr strings hardcoded. Functional but untenable long-term. Unity Localization package integration + CSV export is T-INTEGRATION-002. No player-visible impact at 80% parity but required for App Store submissions. Effort: 8-12h dedicated sprint.
