# N27 — W2-1 Castle HP Verify Post-N12 (Q9-4)

## Summary
Static audit reveals **CRITICAL ISSUE**: LevelData assets store hardcoded `castleHPOverride` values that contradict N12 fix. LevelData.CastleHP property correctly applies formula when `overrideCastleHP=true`, but all production levels have overrides enabled with stale values.

## Root Cause
LevelData.cs line 70–72 implements fallback logic:
```csharp
public int CastleHP => overrideCastleHP
    ? castleHPOverride
    : BalanceConfig.Get().CastleHPFor(world, level);
```

**All production levels have `overrideCastleHP=true`** with frozen values from pre-N12 era. The N12 fix to `LevelDifficultyMul[9]=1.30f` is **bypassed** at runtime.

**Current behavior**:
- W1-9: returns 160 (asset) instead of 165 (formula)
- W2-1: returns 130 (asset) instead of 171 (formula)
- W2-9: returns 185 (asset) instead of 222 (formula)
- W3-1: returns 150 (asset) instead of 187 (formula)

## Q9-4 Satisfaction
**Status**: ✗ FAILED — Q9-4 requires formula application per design spec. Asset overrides block this.

## Recommendation (N28 candidate)
Disable `overrideCastleHP` on all production LevelData assets (W1-1 through W*-9) to activate formula path. Or bulk-regenerate all override values using BalanceConfig formula post-N12.
