### Bottleneck #1 prioritaire : MuzzleFlash `new GameObject` / `Destroy`

Chaque tir de tour non-Frost crée un Point Light temporaire via `new GameObject + AddComponent<Light> + Destroy(go)`.
Avec 12 tours à ~1.5 tirs/s = 18 Instantiate+Destroy par seconde = 18 GC allocs + 18 culling overhead par seconde.
Solution minimale : pool de 4–6 lights réutilisables (pattern déjà utilisé ailleurs pour projectiles et VFX).

### Bottleneck #2 : `HasAnimatorParam` per frame

`animator.parameters` retourne une copy array à chaque accès (Unity property). Appelé conditionnellement
mais potentiellement chaque frame pour les ennemis avec animator et `cfg.Speed > 0f`.
Solution : cacher un `bool _hasSpeedParam` dans `Enemy.Init` (one-time check).

### Bottleneck #3 : EnemyPathingSystem.Tick() jamais appelé

Le système de batching + `Parallel.For` conçu pour 100+ ennemis n'est jamais activé.
Mouvement = N `Update()` callbacks séparés = N transform accès non-batché.
Solution : appeler `EnemyPathingSystem.Instance?.Tick()` depuis `LevelRunner.Update()`
ET exclure le mouvement inline dans `Enemy.Update.cs` via `if (IsPathable) return;`
(les ennemis pathables sont déjà traités par le système batch).
