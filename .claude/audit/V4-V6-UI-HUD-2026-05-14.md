Comparaison exhaustive entre **V4** (Phaser/HTML/CSS, `/Users/mike/Work/milan project/src-v3/`, deploy `/v4/`) et **V6** (Unity 6 UI Toolkit, `/Users/mike/Work/crowd-defense/`, deploy `/v6/`).

- **P0** = game-breaking / élément essentiel manquant (gameplay impossible ou très dégradé).

## Écarts (gap list)

### P0 — Game-breaking ou élément essentiel manquant

1. **Encyclopédie (Tours / Ennemis / Perks) absente du HUD V6** — V4 a `#encyclopedia` accessible via bouton 📚 top-right depuis le jeu. V6 a `BestiaryPanel.uxml` mais accessible uniquement depuis le MenuLevelSelect, pas depuis le HUD en jeu, et il manque Tours + Perks (juste Bestiaire). Suggestion : ajouter `btn-encyclopedia` dans la top-bar HUD + Panel agrégé Tours/Ennemis/Perks ou ouvrir BestiaryPanel + TowerStatsPanel + un nouveau PerksPanel.

2. **Hero-bar XP V6 utilise `Hero` comme label statique au lieu du nom héros / portrait** — V4 affiche `👑 Lv N` avec emoji couronne. V6 affiche juste "Hero" + "Lv 1" sans portrait. Suggestion : intégrer `HeroPortrait.uss` au hero-panel ou bind `HeroPortraitController` ; afficher le skin/portrait équipé.

3. **Boss-rush et Daily Challenge level tiles manquent dans le LevelSelect V6** — V4 worldmap : `endless` tile (∞ + Best:VN + leaderboard top 3) + `boss_rush` tile (BOSS) + `daily` tile (📅 streak). V6 a `DailyChallengeModal.cs` mais pas de tile dédiée pour boss_rush / endless dans WorldMap.uxml. Suggestion : ajouter ces 3 modes spéciaux dans `LevelSelectController` ou `WorldMapController`.

4. **Run Mode (3 écoles + map procédurale) absent V6** — V4 a un mode Run complet (`ui/RunMode.js`) avec 3 écoles Feu/Givre/Maçonnerie, 7 actes, nodes combat/elite/mystery/shop/rest/boss, perks de run, abandon/reprendre, victory/defeat avec gems. V6 n'a que `RunMap.uxml` placeholder. Suggestion : implémenter `RunModeController` complet + brancher au menu principal "NEW GAME" et écran d'abandon/résultat.

5. **Magic Schools / écoles selection screen absent V6** — V4 lance une popup 3 cards (Feu / Givre / Maçonnerie) avec icon + desc + perks exclusifs au début de chaque run. V6 n'a aucun équivalent. Suggestion : nouveau `SchoolSelectScreen.uxml` + controller.

6. **Wave UI : pas de "Vague N arrive" banner / "Vague N terminée" banner V6** — V4 a `#wave-start-banner` (orange "⚠ Vague N arrive") et `#wave-clear-banner` (vert "✓ Vague N terminée") qui slide-in 1.2-1.5s à chaque wave. V6 a `WaveBannerController.cs` mais le UXML `wave-cleared` row UXML est absente, et il n'y a pas de banner "wave-start" simple. Suggestion : ajouter 2 labels banner top-center + binding sur `OnWaveStart`/`OnWaveCleared`.

7. **V6 — pas de wave countdown affiché entre vagues** — V4 a `#wave-countdown` "Vague N+1 dans Xs". V6 a `wave-launch-pill` (countdown du skip bonus) mais pas un countdown explicite "dans Xs" pour auto-start si l'auto-start fail-safe (60s) est implémenté. Avec le bouton manuel ce serait redondant mais utile en mode novice. Suggestion : afficher pill quand mode auto-start, masquer en manual.

8. **Result screen sans bouton "Boutique" V6** — V4 result modal a 4 boutons : Carte / Boutique / Rejouer / Niveau suivant. V6 `LevelSummary.uxml` n'a que Recommencer / Suivant / Carte du monde (pas de Boutique). Le shop méta est inaccessible directement post-victoire. Suggestion : ajouter `summary-btn-shop`.

9. **Skin shop V6 incomplet** — V4 a 12+ skins répartis en 3 onglets (Hero / Castle / VFX) avec déblocage par défaut/achievement/boss-drop/purchase. V6 a `SkinPicker.uxml` mais Shop.uxml séparé et la trinité onglet (Hero/Castle/VFX) n'apparaît pas dans Shop.uss visible. Suggestion : vérifier que les 3 catégories sont rendues + parité de count de skins.
