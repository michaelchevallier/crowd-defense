## 🏗️ Root Causes of Gaps

### 1. Shaders (Animated terrain)
- **V4** : Shaders.js custom GLSL + Three.js ShaderMaterial → animated lava/water frames (4×4 tile animation cycle)
- **V6** : Deferred to Phase 3 (no allocated bandwidth in POC). Would require:
  - **Shader Graph** for animated materials (URP-compatible)
  - **Texture streaming** : 4 keyframe PNGs per theme (lava, water)
  - **Integration** : MapRenderer.cs apply to terrain mesh

### 2. Weather system
- **V4** : Weather.js emits particles (spores/sand/embers/confetti) based on theme + particle pool ticks
- **V6** : Not started. Would need:
  - **Theme-driven particle config** (SpawnRate, Color, Lifetime, Velocity)
  - **Particle emitter placement** (overhead camera frustum edge)
  - **Integration** : LevelRunner loop tick weather system

### 3. Damage popups
- **V4** : makeDamageSprite() renders damage value to canvas, caches texture, spawns Sprite3D billboard
- **V6** : Basic text labels via Unity UI TextMeshPro → less visual punch
- **Fix** : Spawn floating 3D quads with damage numbers (shader-tinted, same canvas approach)

### 4. Boss shaders (Jellyfish/Hologram)
- **V4** : createJellyfishMaterial() + createHologramMaterial() in Shaders.js
- **V6** : Code ported to Shaders.cs, but **no play-mode validation** → likely untested
- **Fix** : Integrate into Enemy.cs Boss phase logic, validate Kraken + AI Hub visually

### 5. Particle pooling
- **V4** : Particles.js maintains 400-particle pool, recycles via cursor
- **V6** : JuiceFX.cs covers camera shake/flash/slowmo; full particle pool pending
- **Fix** : Implement VfxPool.cs (inherit from ObjectPool<Particle>), feed death explosions + hit bursts
