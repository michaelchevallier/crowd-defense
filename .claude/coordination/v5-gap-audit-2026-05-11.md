## 3. Top 10 MANQUANT par impact gameplay

Classés du plus impactant au moins :

1. **RunMap.js (168 LOC)** — Roguelike map graph (act 1-3, node types combat/rest/shop/mystery/boss, edges aléatoires, swarmMul). C'est l'ossature du run flow. Sans ça pas de roguelike loop, juste level select linéaire.

2. **PathTiles.js (600 LOC)** — Streams (water/lava traversées) + bridges textures animées. Gros morceau visuel. Sans ça les paths W3 desert / W4 volcan / W8 submarin manquent de signature visuelle.

3. **Shaders.js Water/Lava/Portal/Hologram/Kelp/Jellyfish/Starfield/SmokeTrail (445 LOC)** — 8 shaders custom V5 pas portés (seul toon de base et outline présents). Bloque l'identité visuelle des 6+ thèmes (espace, submarin, volcan, foret, apocalypse).

4. **PathVariant.js (124 LOC)** — 3 path layouts par world pour variabilité. Sans ça, replay value baisse (joueur voit toujours même map par level).

5. **MapValidator.js (283 LOC)** — Validate grid sanity (portals, castles, walkable connectivity). Utile dès qu'on ouvre level edition aux outils ou aux modders. Pas bloquant gameplay mais bloque pipeline Editor.

6. **Cutscene .asset peuplé (0/10 world intros)** — Schéma `CutsceneDef.cs` + registry existent mais aucun asset cutscene créé. World 1-10 intros narratives manquantes → onboarding narratif zéro.

7. **Modifier .asset peuplé (0/8 V5 entries)** — Curses/blessings runtime (dragon_breath -30% range, ancestral_fog -50% hero range, magnetic_storm projectile sway, etc). Schéma OK mais loop curse system non instancié.

8. **Event .asset peuplé (0 entries vs 12 V5 RUN_EVENTS)** — ash_merchant / wandering_knight / ancient_forge / etc. Bloque le système de roguelike encounters. EventSystem.cs + EventChoiceOverlay.cs existent mais boucle vide.

9. **School Select Modal + Run Summary screens (depuis RunMode.js)** — Démarrer un run sans choisir d'école → tout part faux. Et un run sans victory/defeat screen → pas de feedback explicit (joueur ne sait pas qu'il a gagné/perdu).

10. **SchoolDef.cs SO (absent)** — V5 5 schools (Glaciation, Pyromancie, Forteresse, Cristal, Combustion, MursPierre) traités comme perks subset. Pas un blocker mais discipline manquante pour évolutions futures (école = identité run).
